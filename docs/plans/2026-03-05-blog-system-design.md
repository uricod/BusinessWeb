# Blog System Design

## Date: 2026-03-05

## Overview
Add an SEO-optimized blog to theacropora.com. Blog content is generated by SEOmachine as markdown files, dropped into the repo, and deployed statically via Cloudflare Pages.

## Key Constraint
Homepage (`/`) stays exactly as-is. No blog links visible until user scrolls down to navbar. Direct links to `/blog` or `/blog/[slug]` work for SEO traffic.

## Architecture

### Routing
- `/` -- Homepage (unchanged)
- `/blog` -- Blog listing page (all posts, newest first)
- `/blog/[slug]` -- Individual article page

### Content Source
Markdown files with YAML frontmatter in `src/content/blog/`. Each `.md` file = one blog post.

Frontmatter schema:
```yaml
---
title: "Article Title"
description: "Meta description for SEO"
date: "2026-03-10"
keywords: ["keyword1", "keyword2"]
author: "TheAcropora"
---
```

### Static Export
Works with `output: "export"`. Next.js generates all blog pages at build time via `generateStaticParams`.

## Navbar Behavior
- **Homepage (`/`)**: Unchanged -- hidden until scroll, anchor links
- **Blog pages**: Always visible. Logo links to `/`. "Blog" link added. "Our Work" and "Contact" point to `/#testimonials` and `/#contact`.

## Blog Listing Page (`/blog`)
- Grid of post cards (title, date, excerpt, read time)
- Matches existing site aesthetic (dark background, gradient accents)
- Proper meta tags

## Article Page (`/blog/[slug]`)
- Rendered markdown with proper typography
- Auto-generated table of contents from headings
- Per-article meta tags (title, description, keywords, Open Graph)
- CTA at bottom linking to contact section
- "Back to Blog" navigation
- Article structured data (JSON-LD)

## Markdown Rendering
- `gray-matter` for frontmatter parsing
- Lightweight markdown renderer (react-markdown or compile at build time)
- Support: code blocks, tables, bold, lists, headings
- Plain markdown, no MDX

## SEO Features
- Per-article meta title, description, keywords
- Open Graph tags for social sharing
- Canonical URLs
- Article structured data (JSON-LD)
- Clean URL slugs (`/blog/primary-keyword-phrase`)

## Content Pipeline
1. SEOmachine generates markdown in its output/drafts directory
2. Copy final markdown to `src/content/blog/` in businessweb
3. Commit and push
4. Cloudflare Pages auto-deploys
